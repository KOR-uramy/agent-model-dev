# 02 Action

Need를 충족하기 위한 구체 행동을 정의한다.  
보통 최근 `.ralph/progress.md`의 최신 진행 메모 또는 현재 이터 목표에서 Action을 읽어온다.

**병렬 스레드**: **Core(1~3)**. 트리거는 `01_need.md`에 남긴 미완 체크리스트.

**근거 파일**: 기본은 `.ralph/progress.md`(워크스페이스 루트). 병렬 에이전트 모드에서는 이 파일을 수정하지 않을 수 있으므로, 동일 정보를 커밋 메시지·PR 본문·이슈에도 남긴다. API는 가능하면 progress 마지막 비헤더 줄을 `flow.action`으로 노출한다.

## Checklist (작성자: 01 Need 단계)

- [ ] Need를 충족하기 위한 실행 가능한 Action 1~3개를 적는다.
- [ ] 각 Action이 어느 역할(기획/구현/검증)에서 처리될지 태깅한다.
- [ ] 각 Action이 `RALPH_TASK.md`의 어떤 미완 `[ ]`(또는 섹션)에 대응하는지 연결한다.
- [ ] progress를 갱신할 수 없을 때를 대비해, 대체 근거(브랜치명·커밋 해시·PR 링크 중 1개 이상)를 적어 둔다.
- [ ] `03_capability_business_logic.md`에 넘길 때 필요한 스크립트/모듈 힌트를 한 줄로 적는다.
