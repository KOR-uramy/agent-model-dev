# 01 Need

명시적 요구사항/목표를 정의한다.  
실행 기준은 워크스페이스 루트의 `RALPH_TASK.md`에 있는 미완 `[ ]` 항목이다. Need(1단계)만 예외적으로 **직전 단계 체크리스트가 아니라** 루프·과제 입력 자체가 트리거이다.

`GET /api/ralph/layer-flow`의 `flow.need`는 위 파일에서 **첫 번째 미완 체크** 한 줄을 요약해 보여 준다.

## Checklist (전단계 입력 없음: 시작점)

- [ ] 현재 루프의 최우선 Need를 한 줄로 명시한다.
- [ ] Need가 측정 가능한 성공 조건(`RALPH_TASK.md` Success Criteria의 동일 주제 `[ ]`)과 연결되는지 확인한다.
- [ ] 이번 사이클에서 다룰 Success Criteria 항목을 최소 1개 지목하고, 완료 시 검증 방법(빌드/스모크 등)을 적어 둔다.
- [ ] Core 스레드(1~3) 핸드오프: 02 Action에 넘길 때 Need에서 직접 파생된 행동 후보가 있는지 스스로 점검한다.
