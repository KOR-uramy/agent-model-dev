# Agent job5 — 수익화 문구 정합성 검토 (UI · README · `.env.example`)

## 요약

- **결제 구현**은 토스페이먼츠 v2 **단건 결제** 한 건으로 워크스페이스 `subscriptionStatus`를 `active`로 바꾸는 형태이며, UI·`apps/open-graze/.env.example`·`apps/open-graze/README.md`는 대체로 “구독(단건)·결제위젯·금액 환경변수” 서술에 맞는다.
- **동종 SaaS**(예: PostHog, Linear, Stripe Billing형 제품) 대비 **플랜 다단계·가격표·청구서·환불·무료 체험**은 제품·문서 어디에도 정의되어 있지 않다. 아래 `- [ ]`를 후속 작업 후보로 둔다.
- **문구 불일치 1건**: 앱 대시보드 링크는 **「구독 · 결제」**, README 단계 설명은 **「구독 (토스)」**였다 → `apps/open-graze/README.md`를 UI 표기에 맞춤.

## 변경 사항

| 파일 | 내용 |
|------|------|
| `apps/open-graze/README.md` | 토스 결제 절차에서 대시보드 진입 링크 표기를 UI와 동일하게 정리 |
| 본 리포트 | 동종 대비·갭 `- [ ]` 목록 추가 |

## 동종 대비(벤치마크 관점)

- B2B SaaS는 보통 **공개 가격/플랜 페이지**, **결제 수단 실패 시 재시도·카드 갱신 안내**, **인보이스·영수증**, **환불·취소 정책 URL**, **무료 체험 기간**을 랜딩 또는 설정·청구 UI에 둔다.
- 본 앱은 **워크스페이스당 단일 금액(`TOSS_SUBSCRIPTION_AMOUNT_KRW`, 미설정 시 최소 100원)** + 위젯 결제·실패 리다이렉트 페이지 수준으로, 위 항목 대부분이 **의도적 미구현**이면 문서에 “미제공”을 명시하는 편이 사용자 기대와 충돌을 줄인다.

## 갭 체크리스트 (`- [ ]` 후속)

- [ ] **플랜** — UI·README에 “플랜”이 없고 `subscriptionStatus`만 노출된다. 다티어 플랜이 없다면 랜딩·README에 **「단일 상품(워크스페이스 활성화)」** 등으로 명시한다.
- [ ] **가격** — 결제 페이지 본문에 **원화 금액·부가세 포함 여부**가 고정 카피로 없고(위젯에만 의존), README·`.env.example`는 기본 100원만 언급한다. 운영 금액과 동일한 **한 줄 설명**을 UI 또는 README 한 곳에 둔다.
- [ ] **결제 실패** — `billing/fail`은 토스 `message`·`code`를 보여준다. README에는 **실패 URL·사용자 안내(재시도·고객센터)** 가 없다. `docs/ux-friction-scenarios.md`의 결제 오류 카피 항목과 합친다.
- [ ] **청구** — 인보이스·결제 내역·다운로드 UI 및 문서가 없다. 필요 없으면 “제공하지 않음”을 정책 문서에 적는다.
- [ ] **환불** — 정책·절차·연락처가 없다. 토스/내부 정책에 맞는 **한 페이지 또는 README 단락**을 추가한다.
- [ ] **무료 체험** — 체험 기간·제한 문구가 없다. `inactive`가 곧 무료 티어라면 그렇게 **명시**하고, 아니면 trial 정책을 정의한다.
- [ ] **성공 피드백** — `?billing=success`로 리다이렉트되나 워크스페이스 상세에서 **성공 토스트/배너**가 없어 사용자이 혼동할 수 있다.
- [ ] **Stripe 레거시** — Checkout은 `?billing=cancel` 등 다른 쿼리를 쓴다. 토스 실패 페이지와 **용어·플로**를 README에서 구분해 적는다.
- [ ] **루트 `.env.example`** — 플랫폼 self-test만 있고 **결제 변수는 없음**. “앱 쪽은 `apps/open-graze/.env.example`”을 한 줄이라도 넣을지(병합 정책과 합의 후) 정한다.
- [ ] **루트 `README.md`** — “Google 로그인” 등과 실제 Credentials 플로가 어긋날 수 있다(수익화 범위 밖이나 신뢰에 영향). 별도 문서 이슈로 분리한다.

## 주의 (gotchas)

- **병렬 규칙**: 루트 `README.md`, 루트 `.env.example`, `RALPH_TASK.md`, `.ralph/progress.md`는 **이번에 수정하지 않음**(충돌·오케스트레이터 담당).
- **용어** — 코드·주문명은 “구독”, 구현은 **단건 결제**; 외부 커뮤니케이션에서 “정기 구독”으로 오해되지 않게 할지 검토한다.
